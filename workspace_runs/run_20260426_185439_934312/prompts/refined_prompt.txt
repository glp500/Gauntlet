# Task Objective
Analyze the provided sales and inventory dataset to derive meaningful insights, produce a clear summary, and visualize the findings with at least one explanatory figure.

# Input Dataset Summary
The input is a single CSV file named `Warehouse_and_Retail_Sales.csv`.
*   **Size:** 307,645 rows.
*   **Columns:** `YEAR`, `MONTH`, `SUPPLIER`, `ITEM CODE`, `ITEM DESCRIPTION`, `ITEM TYPE`, `RETAIL SALES`, `RETAIL TRANSFERS`, `WAREHOUSE SALES`.
*   **Data Types:** Numerical columns include `RETAIL SALES` (float64), `RETAIL TRANSFERS` (float64), and `WAREHOUSE SALES` (float64).

# Assumptions From Schema
1.  The analysis will focus on time-series data (based on `YEAR` and `MONTH`).
2.  The analysis can involve calculating aggregated metrics (sums, averages, trends) across sales, transfers, and warehouse values.
3.  The data quality is sufficient for standard pandas analysis (no missing values implied, although this should be checked during processing).

# Required Outputs
1.  A clear textual summary of the dataset's key findings.
2.  At least one figure (e.g., line plot, bar chart) that visually explains a significant result derived from the analysis.

# Code Constraints
1.  **File Access:** Code must load the data from the provided CSV file.
2.  **Libraries:** Only `pandas` and `matplotlib` are permitted for data manipulation and visualization.
3.  **Execution Environment:** No external network access, shell commands, or external APIs are allowed.
4.  **Module Structure:** Generated code must implement logic within the following files: `data_loader.py`, `preprocessing.py`, `analysis.py`, and `figures.py`.
5.  **Logic:** All analysis and visualization logic must be explicit, readable, and self-contained.
